# 🌀 Site Replicator Pro (Skill)

Esta es la habilidad más potente del `frontend-engineer`. Permite diseccionar cualquier sitio web "World-Class" y replicar su estética y movimiento.

## 📋 Playbook de Ejecución (Las 4 Fases de Élite)

### FASE 1 — RECONOCIMIENTO (Investigación Profunda)
1. **Scrapeo del HTML Fuente:** Usar `curl` con User-Agent moderno para descargar el `page_source.html`.
2. **Extracción de Assets:** Listar URLs de imágenes reales, tipografías, variables CSS e iconos.
3. **Análisis Visual:** Documentar cada sección (navbar, hero, footer) basándose en la captura comparándola con el código extraído.

### FASE 2 — MAPA DE SECCIONES
- Crear un mapa visual ordenado de TODAS las secciones antes de escribir código.
- Definir fondos, layouts y textos literales (copiados exactamente, sin paráfrasis).

### FASE 3 — CONSTRUCCIÓN PIXEL-PERFECT
- **Estructura Semántica:** HTML5 (nav, main, section, footer).
- **CSS:** Uso de `:root` para colores exactos, `backdrop-filter` para transparencias y `sticky` para navbars.
- **Imágenes:** Usar URLs reales absolutas extraídas del fuente.
- **Tipografía:** Copiar el font-stack exacto y pesos originales.
- **Responsive:** Implementar breakpoints específicos (1200px, 1024px, 768px, 480px).

### FASE 4 — QA CHECKLIST (Revisión Final)
- [ ] ¿Todas las secciones están en el orden vertical correcto?
- [ ] ¿Los textos son literales al 100%?
- [ ] ¿Las imágenes cargan desde URLs reales?
- [ ] ¿El diseño es responsive y no rompe en mobile?
- [ ] ¿El código es limpio, sin trackers o JS innecesario?

## 🚀 Entregable Final
Un archivo `index.html` (o componente React equivalente en Syrtix) que sea visualmente indistinguible del original.

## ⚠️ Reglas Éticas de Syrtix
- Usamos la clonación para **estudio, inspiración y mejora**. 
- Todo diseño clonado debe ser "Syrtix-izado" para que sea una versión mejorada y no un plagio directo.
